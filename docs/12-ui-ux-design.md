# UI/UX 설계 문서

## 1. UI 설계 원칙

### 원칙
1. **정보 계층:** 가장 중요한 정보(보스 HP, 데미지)가 항상 보임
2. **한 손 조작:** 모바일 세로모드 대응 (하단 60%에 인터랙션 집중)
3. **직관적 피드백:** 카드 매칭/족보 완성 시 즉각적 시각/청각 피드백
4. **한국적 미학:** UI 프레임에 전통 문양, 먹물 테두리 적용

## 3. 인터랙션 설계

### 카드 플레이 플로우
```
[손패 카드 탭]
    │
    ├─ 카드 확대 표시 (0.2초)
    │
    ├─ 바닥에 같은 월 있으면 하이라이트
    │
    ├─ [확인 탭] 또는 [드래그 → 바닥]
    │     │
    │     ├─ 매칭 0개: 카드가 바닥으로 슬라이드
    │     ├─ 매칭 1개: 두 카드가 획득패 영역으로 이동
    │     ├─ 매칭 2개: "선택하세요" + 2장 하이라이트
    │     └─ 매칭 3개: 4장 모두 획득패로 이동 (쓸 연출)
    │
    └─ [취소]: 카드 돌아감 (확인 전까지 취소 가능)
```

### 뒤집기 플로우
```
[자동 진행]
    │
    ├─ 뽑기패에서 1장 뒤집기 (카드 플립 애니메이션)
    │
    ├─ 0.5초 대기 (카드 확인 시간)
    │
    └─ 매칭 자동 실행 (2장 매칭 시만 선택)
```

### Go/Stop 플로우
```
[족보 완성 감지]
    │
    ├─ 족보 완성 연출 (텍스트 + 이펙트)
    │
    ├─ 점수 계산 표시 (칩 × 배수 = 최종)
    │
    ├─ Go/Stop 패널 등장
    │     │
    │     ├─ Go 버튼: 리스크 정보 표시
    │     │     └─ [탭] → 계속 진행 + 배수 증가 연출
    │     │
    │     └─ Stop 버튼: 확정 점수 표시
    │           └─ [탭] → 점수 정산 연출 → 결과 화면
    │
    └─ 5초 타이머? (긴장감 요소, 선택사항)
```

## 4. 피드백 & 연출

### 시각적 피드백
| 이벤트 | 연출 |
|--------|------|
| 카드 매칭 성공 | 두 카드 사이에 빛줄기, 화면 미세 진동 |
| 쓸 (3장 매칭) | 화면 전체 플래시, "쓸!" 텍스트 |
| 족보 완성 | 족보명 대형 텍스트 + 먹물 퍼짐 이펙트 |
| Go 선택 | 화면 붉어짐, 도깨비불 증가, 심장박동 이펙트 |
| Stop 데미지 확정 | 데미지 숫자 카운트업 → 보스 HP 바 감소 애니메이션 |
| 보스 기믹 발동 | 화면 흔들림 + 보스 대사 + 전용 이펙트 |
| 부적 발동 | 부적 아이콘 빛남 + 효과 수치 팝업 |
| 승리 | 금빛 파티클 + 팡파레 |
| 패배 | 화면 어두워짐 + 균열 이펙트 |

### 데미지 카운트업 연출
```
칩: 120 → [먹물 스탬프 효과]
  ×
배수: 3 → [도깨비불 이펙트]
  =
데미지: [숫자가 빠르게 올라감] 360! → [보스 HP 바에 타격 이펙트]

(부적 발동 시)
→ 피의 맹세 발동! +8 배수
→ 120 × 11 = 1320 DMG!!! → [화면 흔들림 + 보스 HP 바 크게 감소]
```

## 5. 접근성 (Accessibility)

### 색각 이상 대응
- 카드 타입을 색상만이 아닌 아이콘으로도 구분
  - 광: ★ / 띠: ═ / 열끗: ◆ / 피: ●
- 색맹 모드: 대비 강화 팔레트 제공

### 텍스트 크기
- 기본 폰트: 최소 14pt (모바일 기준)
- 점수/배수: 최소 20pt
- 옵션에서 텍스트 크기 조절 가능 (100%/125%/150%)

### 조작
- 터치: 탭 + 드래그 지원
- 마우스: 클릭 + 드래그 + 우클릭(카드 상세)
- 키보드: 방향키 카드 선택, Enter 확인, Esc 취소

### 속도 조절
- 연출 속도: 느리게/보통/빠르게 (옵션)
- "빠른 진행" 토글: 모든 연출 스킵
