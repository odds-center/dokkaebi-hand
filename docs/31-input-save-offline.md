# 입력 & 저장 & 오프라인 설계

## 1. 입력 원칙: 마우스/터치 전용

### 핵심 규칙
> **모든 게임 조작은 마우스 클릭 또는 터치 한 가지로 가능.**
> 키보드는 사용하지 않음. 게임패드는 향후 Steam Deck 지원 시 추가.

### 조작 맵

| 동작 | 입력 |
|------|------|
| 카드 선택 | 손패 카드 클릭/탭 |
| 카드 내기 | 선택된 카드 클릭/탭 (바닥 매칭 자동) |
| 2장 매칭 시 선택 | 하이라이트된 바닥패 중 하나 클릭/탭 |
| Go 선택 | Go 버튼 클릭/탭 |
| Stop 선택 | Stop 버튼 클릭/탭 |
| 다음 라운드 | "다음 라운드 ▶" 버튼 클릭/탭 |
| 상점 구매 | 상품 클릭/탭 |
| 이벤트 선택 | 선택지 버튼 클릭/탭 |
| 이승의 문 | "문 진입" 또는 "계속" 버튼 클릭/탭 |
| 재시작 | "재시작" 버튼 클릭/탭 |
| 메인 메뉴 | "메인 메뉴" 버튼 클릭/탭 |
| 카드 상세 보기 | 카드 길게 누르기 (0.5초 홀드) |
| 부적 상세 보기 | 부적 아이콘 길게 누르기 |
| 족보 가이드 | "?" 버튼 클릭/탭 (토글) |
| 동료 도깨비 스킬 | 동료 아이콘 클릭/탭 |

### 없는 것
- 키보드 단축키 없음
- 드래그 앤 드롭 없음 (모두 클릭/탭)
- 우클릭 없음 (모바일 호환성)
- 더블클릭 없음 (실수 방지)

### UI 터치 영역 가이드
```
최소 터치 영역: 44×44 px (Apple HIG 기준)
카드 크기: 80×120 px (충분)
버튼 크기: 최소 250×50 px
버튼 간 간격: 최소 16 px (오터치 방지)
```

## 2. 저장 시스템: 로컬 + Steam 이중 저장

### 핵심 규칙
> **네트워크 완전 불필요.** 오프라인에서 100% 동작.
> Steam 클라우드는 Steam 클라이언트가 백그라운드에서 동기화 (게임이 직접 네트워크 호출 안 함).
> 인터넷 없어도 로컬 저장으로 완전 동작.

### 저장 구조
```
[게임]
  │
  ├─ 로컬 파일 저장 (1차, 항상)
  │   위치: Application.persistentDataPath/dokkaebi_meta.json
  │   백업: dokkaebi_meta.json.bak (이전 세이브)
  │
  └─ Steam 클라우드 저장 (2차, Steam 있을 때만)
      위치: Steam Remote Storage
      자동 동기화: Steam 클라이언트가 처리

[불러오기]
  │
  ├─ 로컬 & Steam 둘 다 확인
  ├─ 더 최신(Timestamp) 데이터 선택
  └─ 선택되지 않은 쪽을 최신으로 동기화
```

### 저장 시점 (자동)
| 시점 | 트리거 |
|------|--------|
| 관문 클리어 | GameState → PostRound |
| 상점 퇴장 | GameState → Shop 종료 |
| 이승의 문 선택 | GameState → Gate |
| 게임 오버 | GameState → GameOver |
| 앱 종료 | OnApplicationQuit |
| 앱 백그라운드 | OnApplicationPause (Steam Deck 등) |

### 수동 저장 없음
- 로그라이크 특성: 되돌리기 방지
- 자동 저장만 존재
- "세이브 슬롯"도 1개 (현재 진행만)
- 메타 데이터(영구 강화, 업적 등)는 항상 유지

### 저장 데이터 구분
| 데이터 | 런 종료 시 | 영구 |
|--------|----------|------|
| 윤회/관문 위치 | 초기화 | - |
| 목숨/엽전 | 초기화 | - |
| 부적 장비 | 초기화 | - |
| 영혼 조각 | - | ✅ 유지 |
| 영구 강화 레벨 | - | ✅ 유지 |
| 업적 | - | ✅ 유지 |
| 동료 도깨비 | - | ✅ 유지 |
| 카드 강화 등급 | 1단계 하락 (윤회) | ✅ 유지 |
| 통계 | - | ✅ 유지 |

## 3. 오프라인 완전 동작

### 네트워크가 필요한 것: 없음

| 기능 | 네트워크 필요? |
|------|-------------|
| 게임 플레이 | ❌ 불필요 |
| 세이브/로드 | ❌ 불필요 (로컬 파일) |
| Steam 업적 | ❌ Steam 클라이언트가 오프라인 캐싱 |
| Steam 클라우드 | ❌ Steam이 온라인 시 자동 동기화 |
| 리더보드 | ⚠️ 업로드 시만 (오프라인 점수는 로컬 저장 후 온라인 시 전송) |
| DLC | ❌ Steam DRM으로 로컬 검증 |
| 게임 업데이트 | ⚠️ 다운로드 시만 |

### Steam 오프라인 모드 대응
```
Steam 오프라인 모드에서:
1. 게임 정상 실행 ✅
2. 로컬 세이브 정상 ✅
3. 업적 오프라인 캐싱 → 온라인 시 동기화 ✅
4. 클라우드 세이브 보류 → 온라인 시 동기화 ✅
5. 리더보드 보류 → 온라인 시 업로드 ✅
```

## 4. 세이브 파일 경로

### 플랫폼별 저장 위치
| 플랫폼 | 경로 |
|--------|------|
| Windows | `%USERPROFILE%/AppData/LocalLow/[Company]/DokkaebiHand/` |
| macOS | `~/Library/Application Support/[Company]/DokkaebiHand/` |
| Linux | `~/.config/unity3d/[Company]/DokkaebiHand/` |
| Steam Deck | 위와 동일 (Linux) + Steam 클라우드 |

### 파일 목록
```
dokkaebi_meta.json      — 메인 세이브 (자동)
dokkaebi_meta.json.bak  — 백업 (이전 세이브)
```

### 세이브 데이터 크기
- 예상: 5~20 KB (JSON, 텍스트)
- Steam 클라우드 할당: 1 MB (충분)

## 5. 세이브 손상 방지

### 방어 전략
1. **쓰기 전 백업**: 저장 전 기존 파일을 .bak으로 복사
2. **읽기 실패 시 백업 복구**: 메인 파일 손상 시 .bak에서 복원
3. **Steam 이중화**: 로컬 손상 시 Steam 클라우드에서 복구
4. **버전 호환**: SaveData.Version 필드로 마이그레이션 지원
5. **Atomic write 아님**: 파일 시스템 한계상 완벽하지 않으나, 백업으로 보완
