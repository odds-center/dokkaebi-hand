# 저승 장터 & 이벤트 시스템 설계

## 1. 저승 장터 시스템

### 저승 장터 등장 타이밍
- 매 보스전 승리 후 1회 등장
- 총 4회 저승 장터 방문 (1~4층 클리어 후)
- 5층(염라대왕) 전에는 저승 장터 없음

### 저승 장터 구성
```
┌─────────────────────────────────────────┐
│              저승 장터                    │
│                                         │
│  [부적 1]     [부적 2]     [부적 3]      │
│  일반 40엽    희귀 100엽   ???           │
│                                         │
│  [소모품 1]   [소모품 2]                  │
│  패 팩 40엽   체력회복 75엽              │
│                                         │
│  [부적 제거: 50엽]                       │
│                                         │
│  보유 엽전: 150                          │
│  [다음 층으로]                            │
└─────────────────────────────────────────┘
```

### 저승 장터 라인업 규칙

**부적 슬롯 (3개)**
- 슬롯 1: 일반 등급 확정 (30~50엽)
- 슬롯 2: 일반 or 희귀 (50~120엽)
- 슬롯 3: 희귀 or 전설 (100~300엽) — 3층 이후 전설 등장 가능

**소모품 슬롯 (2개)**
| 소모품 | 효과 | 가격 |
|--------|------|------|
| 패 팩 (소) | 다음 라운드 시작 시 손패 +2장 | 40엽 |
| 패 팩 (대) | 다음 라운드 시작 시 손패 +4장 | 80엽 |
| 체력 회복 | 목숨 +1 (최대 3) | 75엽 |
| 저주 해제부 | 저주 부적 1개 제거 | 100엽 |
| 감정부 | 다음 저승 장터에서 전설 부적 등장 확정 | 60엽 |

**부적 제거**
- 50엽으로 장착 중인 부적 1개 제거
- 저주 부적은 제거 불가 (저주 해제부 필요)
- 슬롯 확보를 위한 전략적 선택

### Love2D 프로토타입 구현 현황

- 저승 장터는 `talisman_database.lua`(20종)에서 직접 부적 목록을 가져옴
- 이미 보유 중인 부적은 상점 목록에서 자동 제외
- 저주 부적은 상점에 등장하지 않음
- 등급별 고정 가격 적용: 일반 = 40엽전, 희귀 = 70엽전, 전설 = 120엽전
- 기존 설계의 슬롯별 등급 규칙 대신 단순화된 랜덤 선택 사용

### 이벤트 — Love2D 프로토타입 구현 현황

기존 설계 10종 중 5종을 구현. **모든 선택이 다음 보스전에 직접 영향**을 줌:

| 이벤트 | 선택지 | 보스전 영향 |
|--------|--------|------------|
| 운명의 갈림길 | 붉은 문 | 칩 +25 + **다음 보스 기믹 봉인** |
| | 푸른 문 | 배수 +1 + **다음 보스 HP -15%** |
| | 되돌아간다 | 체력 +1 |
| 저승 방랑자 | 도와준다 | 칩 +15 |
| | 소지품 뒤진다 | +50냥 + **다음 보스 HP +20%** |
| 도깨비불 시험 | 도깨비불(정답) | 배수 +0.5 |
| | 그림자(오답) | 체력 -1 |
| 삼도천 강가 | 기도 (-40냥) | **다음 판 손패 +3** |
| | 동전 던지기 | 50% +100냥 / 50% 전액 소멸 |
| 과거의 도전자 | 패를 가져간다 | 칩 +20, 배수 +0.3 |
| | 쉬어간다 | 체력 +2 |

구현 메커니즘:
- `S._next_boss_hp_mult`: 다음 보스 HP 배율 (0.85 = -15%, 1.2 = +20%)
- `S._next_boss_gimmick_seal`: 다음 보스 기믹을 "none"으로 덮어씀
- 보스 생성 시(`start_realm`) 이벤트 효과 자동 적용 후 플래그 초기화

### 저승 장터 주인: 귀신 상인

**대사 예시:**
- 입장: "어서 와, 살아있는 손님은 오랜만이야."
- 구매: "좋은 눈이군. 잘 골랐어."
- 비쌈: "비싸다고? 목숨값에 비하면 싼 거지."
- 엽전 부족: "돈이 없으면 팔 것도 있잖아... 목숨이라든가."
- 퇴장: "또 와. 살아있다면 말이지. 히히."

## 2. 이벤트 시스템

### 이벤트 등장 타이밍
- 1층, 2층 클리어 후 상점 다음에 이벤트 1회
- 총 2회 이벤트 (MVP 기준)
- Alpha에서 3~4회로 확장

### 이벤트 설계 원칙
1. **항상 의미 있는 선택:** 정답이 없는 트레이드오프
2. **빌드에 영향:** 부적/자원/카드에 직접 영향
3. **스토리 강화:** 저승 세계관을 느끼게 하는 연출
4. **짧고 임팩트 있게:** 30초~1분 내 결정

### 이벤트 목록 (10종)

#### E01: 저승 방랑자
```
[어둠 속에서 다른 망자를 발견한다]
"도와줘... 길을 잃었어..."

A. 도와준다 → 감사 보상: 일반 부적 1개 랜덤 획득
B. 무시한다 → 아무 일도 없음
C. 소지품을 뒤진다 → 엽전 30 획득 but 다음 보스 HP +50
```
**설계 의도:** 도덕적 선택. A는 안전 보상, C는 리스크 보상.

#### E02: 귀신 시장 특별 경매
```
[귀신들이 경매를 하고 있다. 물건은 전설 부적.]
"자, 입찰가를 불러봐!"

A. 목숨 1개 → 전설 부적 획득
B. 부적 2개 제거 → 전설 부적 획득
C. 엽전 전액 → 전설 부적 획득
D. 포기 → 체력 +1
```
**설계 의도:** 전설 부적을 얻기 위한 극단적 트레이드오프. 어떤 자원을 버릴 것인가.

#### E03: 운명의 갈림길
```
[두 개의 문이 있다]
왼쪽: 붉은 문. 뜨거운 기운.
오른쪽: 푸른 문. 차가운 기운.

A. 붉은 문 → 다음 보스 HP -50, 손패 -2장
B. 푸른 문 → 다음 보스 HP +50, 부적 슬롯 일시 +1 (이번 층만)
```
**설계 의도:** 낮은 보스 HP vs 강력한 빌드. 현재 상태에 따라 최적 선택이 달라짐.

#### E04: 도깨비불 시험
```
[도깨비불이 떠다니며 수수께끼를 낸다]
"나는 밤에 태어나 낮에 죽는다. 사람들은 나를 두려워하지만, 나 없이는 길을 찾지 못한다. 나는 누구인가?"

A. "도깨비불" → 정답! 희귀 부적 획득
B. "그림자" → 오답. 피 카드 3장 소멸 (획득한 피에서)
C. "달빛" → 오답. 엽전 -30
```
**설계 의도:** 세계관 몰입 + 보상. 정답을 모르는 유저를 위한 부분적 패널티.

#### E05: 삼도천 기도
```
[삼도천 강가에 도착한다. 물결이 반짝인다.]
"기도를 올리면 무언가 응답할 것 같다..."

A. 기도한다 (엽전 -50) → 다음 라운드 첫 드로우가 광 확정
B. 동전을 던진다 (엽전 -20) → 50% 엽전 ×2 / 50% 엽전 소멸
C. 그냥 지나간다 → 아무 일 없음
```
**설계 의도:** 자원 투자형 도박. 광 확정 드로우는 강력하지만 비쌈.

#### E06: 과거의 도전자
```
[벽에 기대앉은 해골. 손에 화투패 한 장을 쥐고 있다.]
"이 사람도 이승으로 돌아가려 했던 걸까..."

A. 패를 가져간다 → 랜덤 광 or 열끗 1장 영구 획득 (덱에 추가)
B. 부적을 가져간다 → 일반 부적 1개 but "과거의 원한" 저주 부적 장착
C. 쉬어간다 → 체력 +1
```
**설계 의도:** 카드/부적 획득 vs 안전한 체력 회복. 저주 리스크.

#### E07: 도깨비의 내기
```
[잡졸 도깨비가 나타나 내기를 제안한다]
"야, 간단한 내기 하나 하자. 동전 뒤집기야."

A. 받아들인다 → 50% 엽전 +100 / 50% 부적 1개 랜덤 제거
B. 거절한다 → 도깨비가 킥킥대며 사라짐 (무효과)
C. 속임수를 쓴다 → 80% 엽전 +100 / 20% 도깨비에게 들켜서 목숨 -1
```
**설계 의도:** 순수 도박 이벤트. 플레이어의 리스크 성향 테스트.

#### E08: 저승꽃밭
```
[붉은 피안화(彼岸花)가 피어있는 들판]
"꽃잎에서 이상한 힘이 느껴진다..."

A. 꽃을 꺾는다 → 다음 3턴간 모든 배수 +2 (일시 버프)
B. 꽃씨를 모은다 → 엽전 +40
C. 지나간다 → 없음
```
**설계 의도:** 일시적 파워업 vs 안전한 자원. 다음 보스전 직전에 등장하면 A가 강력.

#### E09: 거울 연못
```
[연못에 비친 자신의 모습이... 다르다.]
"연못 속의 나는 다른 선택을 했다..."

A. 손을 넣는다 → 현재 부적 전체를 같은 수의 랜덤 부적으로 교체
B. 얼굴을 비춘다 → 다음 보스의 기믹 미리 확인 (정보 보상)
C. 돌을 던진다 → 연못이 사라지고 엽전 +20
```
**설계 의도:** 빌드 리롤 vs 정보 vs 소량 자원. 현재 빌드가 마음에 안 들 때 A가 매력적.

#### E10: 저승사자의 제안
```
[저승사자(뱃사공)가 나타난다]
"하나 알려줄까. 다음 놈의 약점을..."

A. 듣는다 (엽전 -100) → 다음 보스의 기믹 무효화 (1라운드만)
B. "공짜로 안 돼?" → 저승사자가 씁쓸하게 웃으며 사라짐
C. "알아서 이기겠다" → 다음 보스전 시작 시 Go 배수 +1 보너스
```
**설계 의도:** 기믹 무효화라는 강력한 효과의 비싼 대가. C는 자신감 보상.

## 3. 이벤트 등장 규칙

### 이벤트 풀 분류
| 카테고리 | 이벤트 | 등장 빈도 |
|---------|--------|----------|
| 스토리 | E01, E06 | 높음 |
| 도박 | E05, E07 | 중간 |
| 트레이드 | E02, E03, E09 | 중간 |
| 퍼즐 | E04 | 낮음 |
| 버프 | E08, E10 | 낮음 |

### 등장 로직
```python
# 의사 코드
def select_event(floor, player_state):
    pool = ALL_EVENTS.copy()

    # 이전에 본 이벤트 제거 (같은 런 내 중복 방지)
    pool -= seen_events

    # 층별 가중치 조정
    if floor == 1:
        boost(pool, "스토리")  # 초반: 세계관 몰입
    elif floor >= 3:
        boost(pool, "트레이드", "버프")  # 후반: 전략적 선택

    # 플레이어 상태 기반 조정
    if player_state.lives == 1:
        boost(pool, E08)  # 체력 회복 기회
    if player_state.yeop < 50:
        boost(pool, E07, E08)  # 자원 획득 기회

    return weighted_random(pool)
```
