# 이미지 생성 프롬프트 — 도깨비의 패 (Flux-dev + ComfyUI)

## 원칙

Flux-dev는 **분위기와 컨셉**이 강하고, 텍스트 생성 능력도 SD 1.5보다 우수하다.
아래 10가지 영역을 다룬다.

| 폴더 | 용도 | 수량 | 결과물 |
|------|------|------|--------|
| `01-backgrounds/` | 게임 배경 | 20 프롬프트 | **바로 게임에 사용** |
| `02-card-illustrations/` | 카드 내부 일러스트 | 48장 | **프레임 합성 후 사용** |
| `03-textures/` | 한지, 먹물, 천 등 + UI 텍스처 | 15종 | **바로 게임에 사용** |
| `04-concept-art/` | 캐릭터/보스 디자인 | 9캐릭터 | **디자인 레퍼런스** |
| `05-calligraphy/` | 서예/캘리그래피 텍스트 | 37종 | **확인 후 사용 or 폰트 합성** |
| `06-game-sprites/` | 보스/동료/NPC/파츠오버레이/세트오라 | 53종 | **배경 제거 후 사용** |
| `07-icons/` | 부적/업적/통화/강화/파츠 아이콘 | 87종 | **다운스케일 후 사용** |
| `08-illustrations/` | 이벤트/튜토리얼 삽화 | 14종 | **바로 게임에 사용** |
| `09-card-extras/` | 카드 뒷면 + 강화 오버레이 | 6종 | **프레임 합성 후 사용** |
| `10-vfx/` | 파티클 6종 + 기믹 이펙트 10종 | 16종 | **알파 변환 후 사용** |
| `11-ui-frames/` | 패널/버튼/대화/배너/카드 프레임 | 25종 | **9-slice 후 사용** |
| `12-hud-icons/` | 생명력/카드/진행/슬롯/점수 HUD | 21종 | **다운스케일 후 사용** |

## 생성 환경

```yaml
Model: Flux-dev (ComfyUI)
Sampler: euler
Scheduler: normal
Guidance: 3.5~4.0
```

### Flux-dev 핵심 규칙
- **네거티브 프롬프트 미사용** — Flux-dev는 네거티브를 지원하지 않음
- **가중치 문법 미사용** — `(keyword:1.3)` 대신 자연어로 강조
- **LoRA는 ComfyUI 노드에서 연결** — 프롬프트에 `<lora:...>` 태그 넣지 않음
- **자연어 문장 서술** — 쉼표 나열보다 완전한 문장이 효과적
- **이미지 넘침 방지** — 모든 요소가 경계 안에 완전히 들어오도록 마진 확보

### 시각 통일성 원칙 (Art Bible)

모든 프롬프트는 아래 규칙을 따라 **게임 전체의 시각적 일관성**을 유지한다.

| 원칙 | 내용 |
|------|------|
| **아트 스타일** | 16-bit 픽셀아트. 모든 에셋에 "pixel art, crisp sharp pixels, no anti-aliasing" 명시 |
| **윤곽선** | 굵고 선명한 검정 아웃라인. 모든 오브젝트에 thick black outlines |
| **채색** | 플랫 컬러 (flat color fill). 그라디언트 없이 면 단위 채색 |
| **팔레트** | 게임 9색 팔레트 기반 (아래 참조). 각 에셋의 고유색은 팔레트 위에 추가 |

**게임 통일 팔레트 (docs/13-art-sound-direction.md 기준):**

| 색상 | Hex | 용도 |
|------|-----|------|
| 먹빛 | `#1A1A2E` | 배경, 어두운 영역, UI 패널 |
| 한지 | `#F5E6CA` | 카드 앞면, 텍스트 배경 |
| 핏빛 | `#C41E3A` | 홍단, 강조, 데미지, 보스 |
| 도깨비불 청 | `#00D4FF` | 도깨비불, 청단, 마법 |
| 금빛 | `#FFD700` | 광, 전설, 승리, 보상 |
| 자주 | `#6B2D5B` | 전설 부적, 저주, 그림자 |
| 흰빛 | `#E8E8E8` | 피 패, 텍스트, 뼈 |
| 도깨비불 초록 | `#39FF14` | 부적 발동, 시너지 |
| 먹물 회색 | `#2D2D44` | 카드 뒷면, 보조 배경 |

**카드 디자인: A 스타일 (전통 화투 80% + 저승 힌트 20%)**

## 게임 해상도 & UI 크기 참조

| 항목 | 크기 (px) | 비고 |
|------|-----------|------|
| Reference Resolution | 1920x1080 | CanvasScaler, 16:9 |
| 게임 배경 | 1920x1080 | 전체 화면 |
| 카드 (UI 표시) | 90x130 | 게임 화면에서 보이는 크기 |
| 카드 (텍스처 생성) | 80x120 | MockupSpriteFactory |
| 카드 (AI 생성) | 270x390 | UI 크기의 3배 |
| 손패/바닥패 영역 | 950x160 | 카드 놓는 영역 |
| 버튼 | 300x55 | UI 버튼 |
| 보스 HP 바 | 600x30 | 보스 체력바 영역 |
| 점수 패널 | 400x80 | 칩/배수 표시 |
| 부적 아이콘 | 48x48 | 코드 생성 |
| 보스 표시 영역 | ~300x300 | 화면 상단 중앙 |
| 동료 표시 영역 | ~120x180 | |

## AI가 하지 않는 것 (코드/수동)

- 카드 프레임 (테두리, 심볼) → `MockupSpriteFactory`
- 부적 아이콘 48x48 → 코드 생성
- UI 버튼/패널 → 코드 생성
- 파티클/VFX 텍스처 → 코드 또는 간단 에디터
- 한글 텍스트 → 폰트 렌더링 (단, 장식 서예는 `05-calligraphy/` 참조)

## 프롬프트 작성 규칙

1. **자연어 문장 서술** — 완전한 문장으로 장면/글씨를 묘사
2. **공통 프리픽스 + 개별 프롬프트** 순서로 결합
3. 각 프롬프트에 Seed, 배치 수, 후처리 방법 명시
4. 같은 카테고리는 **공통 프리픽스를 유지**하여 화풍 통일
5. 주요 배경에는 **2~3가지 프롬프트 변형** 제공 (선택지)
6. 각 프롬프트에 **색감 지시**와 **연출 의도** 포함
7. **이미지 넘침 방지** — 모든 요소가 경계 안에 마진 두고 배치

## 작업 순서 권장

```
1순위: 텍스처 (빠르고 확실한 결과, 자신감 획득)
2순위: 배경 (AI의 최대 강점, 게임 분위기 확정)
3순위: 카드 일러스트 (48장이라 시간 소요, 일관성 주의)
4순위: 카드 추가 에셋 (뒷면 + 강화 오버레이 — 카드 시스템 완성)
5순위: 캘리그래피 (Flux-dev의 텍스트 생성 활용, 후처리 필요)
6순위: 아이콘 (부적/업적/통화/파츠 — 87종 대량 생산)
7순위: 인게임 스프라이트 (보스/동료/NPC — 게임 캐릭터 비주얼)
8순위: 이벤트/튜토리얼 삽화 (14장 — 스토리 몰입)
9순위: VFX 파티클 (6종 — 시각 효과 품질 향상)
10순위: 컨셉아트 (레퍼런스용, 게임에 직접 안 들어감)
```
