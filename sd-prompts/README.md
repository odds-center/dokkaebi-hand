# SD 프롬프트 — 도깨비의 패 (현실적 파이프라인)

## 원칙

SD는 **분위기와 컨셉**이 강하고, **정밀한 규격 에셋**은 약하다.
따라서 아래 5가지 영역에 집중한다.

| 폴더 | 용도 | SD 적합도 | 결과물 |
|------|------|-----------|--------|
| `01-backgrounds/` | 게임 배경 12종 | ★★★★★ | **바로 게임에 사용** |
| `02-card-illustrations/` | 카드 내부 일러스트 48장 (90x130 UI / 80x120 텍스처) | ★★★★☆ | **프레임 합성 후 사용** |
| `03-textures/` | 한지, 먹물, 천 등 8종 + UI 텍스처 5종 | ★★★★★ | **바로 게임에 사용** |
| `04-concept-art/` | 캐릭터/보스 8종 | ★★★★☆ | **디자인 레퍼런스** |
| `05-calligraphy/` | 서예/텍스트 이미지 34종 (타이틀, 족보, 보스, UI) | ★★★☆☆ | **서예 폰트 합성 후 사용** |

## 게임 해상도 & UI 크기 참조

| 항목 | 크기 (px) | 비고 |
|------|-----------|------|
| Reference Resolution | 1920x1080 | CanvasScaler, 16:9 |
| 게임 배경 | 1920x1080 | 전체 화면 |
| 카드 (UI 표시) | 90x130 | 게임 화면에서 보이는 크기 |
| 카드 (텍스처 생성) | 80x120 | MockupSpriteFactory |
| 카드 (SD 생성) | 270x390 | UI 크기의 3배 |
| 손패/바닥패 영역 | 950x160 | 카드 놓는 영역 |
| 버튼 | 300x55 | UI 버튼 |
| 보스 HP 바 | 600x30 | 보스 체력바 영역 |
| 점수 패널 | 400x80 | 칩/배수 표시 |
| 부적 아이콘 | 48x48 | 코드 생성 |

| 동료 표시 영역 | ~120x180 | |

## SD가 하지 않는 것 (코드/수동)

- 카드 프레임 (테두리, 심볼) → `MockupSpriteFactory`
- 부적 아이콘 48x48 → 코드 생성
- UI 버튼/패널 → 코드 생성
- 파티클/VFX 텍스처 → 코드 또는 간단 에디터
- 한글 텍스트 → 폰트 렌더링 (단, 장식 서예는 `05-calligraphy/` 참조)

## 프롬프트 작성 규칙

1. **토큰 순서가 중요** — 앞에 올수록 영향력이 크다
2. **(keyword:1.3)** — 가중치로 핵심 요소 강조
3. **공통 프롬프트 + 개별 프롬프트** 순서로 결합
4. 각 프롬프트에 Seed, 배치 수, 후처리 방법 명시
5. 같은 카테고리는 **공통 부분을 최대한 유지**하여 화풍 통일
6. 주요 배경에는 **2~3가지 프롬프트 변형** 제공 (선택지)
7. 각 프롬프트에 **색감 지시**와 **연출 의도** 포함

## M3 Pro 18GB 기준 설정

```
모델: SD 1.5 (권장) — 빠르고 안정적
LoRA: pixelart-style (weight 0.5~0.75, 용도별 다름)
Sampler: DPM++ 2M Karras
VAE: 기본
MPS 가속: --force-fp16
```

## 작업 순서 권장

```
1순위: 텍스처 (빠르고 확실한 결과, 자신감 획득)
2순위: 배경 (SD의 최대 강점, 게임 분위기 확정)
3순위: 카드 일러스트 (48장이라 시간 소요, 일관성 주의)
4순위: 컨셉아트 (레퍼런스용, 게임에 직접 안 들어감)
5순위: 서예 (SD 생성 + 서예 폰트 합성 필요, 후처리 많음)
```
